Integration mit Angular

Wir haben ein Demoprojekt mit Angular CLI eingerichtet, um Ihnen zu zeigen, wie Sie die Komponenten in ein neu generiertes Angular-Projekt integrieren können. Scrollen Sie nach unten, um die CodeSandbox zu sehen. Diese Beispiele basieren auf angular@12.0.0 und angular-cli@12.0.0.

Installieren Sie das Paket @inovex.de/elements-angular

Wir stellen ein Integration Layer Package für Angular zur Verfügung, damit Sie die elements als Angular-Bibliothek in Dein Angular-Projekt integrieren können. Um es zu installieren, haben Sie folgenden Optionen:

angular-cli
npm
yarn

ng add @inovex.de/elements-angular

Die ino-icons werden automatisch zum Projekt hinzugefügt, wenn Sie angular-cli nutzen.

Wenn Sie npm oder yarn zum Installieren der elements nutzen, müssen Sie die angular.json manuell anpassen. Fügen Sie Folgendes zu Ihren Build-Assets hinzu (projects.<your-project-name>.architect.build.options.assets) und zum Testen (projects.<your-project-name>.architect.test.options.assets).

angular.json

// ...
{
"glob": "**/*",
"input": "node_modules/@inovex.de/elements/dist/inovex-elements/ino-icon",
"output": "/ino-icon"
}
// ...

Importieren Sie das Modul InoElementsModule

Die Komponenten, die wir im Angular-Projekt bereitstellen, sind in Angular-Direktiven innerhalb einer Angular-Bibliothek verpackt. Alle @Input Eigenschaften und @Output Events unserer Komponenten sind enthalten. Dies ermöglicht es der IDE und dem Compiler, alle Typinformationen aus den gegebenen Schnittstellen abzuleiten, was für eine ordnungsgemäße Code-Vervollständigung zwingend erforderlich ist.

Um die elements zu nutzen, müssen Sie das InoElementsModule in eines deiner App Module integrieren. Führen Sie dazu die folgenden Schritte aus:

  1. Importieren Sie das elements Modul
  2. Fügen Sie unser Modul zu den Imports hinzu (Sie können hierfür auch eine InoElementsConfig erstellen)
  3. Fügen Sie das CUSTOM_ELEMENTS_SCHEMA hinzu
app.module.ts

import {BrowserModule} from '@angular/platform-browser';
import {CUSTOM_ELEMENTS_SCHEMA, NgModule} from '@angular/core';
import {InoElementsModule} from '@inovex.de/elements-angular';
import {AppComponent} from './app.component';
@NgModule({
declarations: [AppComponent],
imports: [
BrowserModule,
InoElementsModule.forRoot({}),
],
providers: [],
bootstrap: [AppComponent],
schemas: [CUSTOM_ELEMENTS_SCHEMA],
})
export class AppModule {}

Wenn Sie mehr als ein Modul haben, sollten Sie das InoElementsModule in jedes Modul integrieren, das die inovex-elements verwenden soll. Alternativ dazu können Sie die Komponenten auch in ein gemeinsames Modul integrieren. Auf diese Weise brauchen Sie die Komponenten nur einmal zu importieren (empfohlen).
