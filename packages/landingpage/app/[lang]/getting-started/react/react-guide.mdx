  ### Playground

  Every component has a variety of powerful and unique properties. They are listed and explained in each component
  **Playground**:


  | Name | Description | Type | Default |
  | ---------------- |
  ---------------------------------------------------------------------------------------------------------- |
  ------------------------------------------------------------------------------------------------- | ---------- |
  | `type` | The type of this element (default = text). | `various | text` | `'text'` |
  | `label` | The optional floating label of this input field. | `string` | `undefined`|
  | `outline` | Styles the input field as an outlined element. | `boolean` | `undefined`|
  | `pattern` | The validation pattern of this element. | `string` | `undefined`|
  | `placeholder` | The placeholder of this element. | `string` | `undefined`|
  | `required` | Marks this element as required. | `boolean` | `undefined`|
  | `show-label-hint`| If true, an _optional_ message is displayed if not required, otherwise a \* marker is displayed
  if required.| `boolean` | `undefined`|
  | `value` | The value of this input field. | `string` | `''` |


  <Hint>Hint: These are some of the properties of the `<InoInput>` component.</Hint>

  The Playground allows you to modify these properties and see the result in real time. Additionally, we provide
  examples about the usage of each component with its given properties.

  ## Events

  With the exact same approach, we use **Events**. They emit a certain value when the user does something that triggers
  our Event to fire. In other terms: Our event gives us a **reaction** to an **action** made by our User.
  In case of our `<InoInput>` the action is the user typing and our events reaction contains the typed input.

  <StepIndicator step='6'>Let's give our login form the `valueChange` event of the `<InoInput>` component, so that we
    can store the value typed by the user into our email or password states.</StepIndicator>


  | Event | Description | Type |
  | ----------------- | ------------------------------------------------------------------------------ |
  ---------------------- |
  | **valueChange** | Emits when the user types something in. Contains typed input in event.detail | CustomEvent\
  <string> |
    | inoBlur | Emits when the input field is blurred and validates email input | CustomEvent\
    <void> |
      | inoFocus | Emits when the input field is focused | CustomEvent\
      <void> |


        ```tsx
        <InoInput
          type="email"
          label="email"
          outline
          value={email}
          onValueChange={value => setEmail(value.detail)}
        ></InoInput>

        <InoInput
          type="password"
          label="password"
          outline
          value={password}
          onValueChange={value => setPassword(value.detail)}
        ></InoInput>
        ```

        <Hint>Hint: while in our Playground the Event is called `valueChange`, we name it `onValueChange` inside of our
          `<InoInput>`. The reason for this is that we want to accomodate our naming similar to the react-way of naming
            events (e.g. onClick)</Hint>

        ## Slots

        Many components of the Elements Library have **slots**, that can be filled by other components or your own
        content to further customize its functionality or appearance.

        <StepIndicator step='7'>If we use the slots of the `<InoInput>` we can add an `<InoIcon>` inside our input
          field, while retaining the full usability of the `<InoIcon>`.</StepIndicator>


        | Slot | Description |
        | ------------------- | ---------------------------- |
        | icon-leading | For the icon to be prepended |
        | **icon-trailing** | For the icon to be appended |


        <Hint>Hint: We can choose the position of our slot, that will render the `<InoIcon>` component. Other components
          like the `<InoCard>` allow a `header`, `content`, `footer` approach of positioning its slots.</Hint>

        ```tsx
        <InoInput
          type="password"
          label="password"
          outline
          value={password}
          onValueChange={(value) => setPassword(value.detail)}
        >
          <InoIcon slot="icon-trailing" icon="password"></InoIcon>
        </InoInput>
        ```

        Note that we indicated the position of the slot inside our `<InoIcon>` with `slot="icon-trailing"`.

          ## CSS-Variables

          In order to provide a CSS way of styling the inovex-elements, we sometimes provide custom properties, a.k.a.
          CSS-Variables.
          In the case of the `
          <ino-input>`, there are two variables we can use to change the appearance of our component:
            `--ino-input-line-color` to change the color of the underline and the `--ino-input-label-color` which
            changes the color of the floating label. They can be used like this:

            ```scss
            InoInput {
              --ino - input - line - color: blue;
              --ino-input-label-color: blue;
            }
            ```

    <StepIndicator step='9'> Use Popovers on Components. The `<InoPopover>` component is perfect for additional
      information or hints.</StepIndicator>

    Make sure the `id` of the hosting element and the `for` property of the attached `<InoPopover>` is the same.
      Given the intention or background of your page, it is helpful to change the `placement`(`top`, `right`, `bottom`,
      `left`).
      Use the `interactive` property if you want to interact with the popover content (e.g. button clicks).

      ```tsx
      <InoButton id="sign-up">Sign Up</InoButton>

      <InoPopover for="sign-up" placement="bottom" interactive>
        <div className="styled-popover">
          <InoIcon icon="user"></InoIcon>
          Already have an account? Sign in.
        </div>
      </InoPopover>
      ```

      <StepIndicator step='10'> Toggle Password Visibility. By clicking on the ino icon lock, we can toggle the
        visibility of our password by changing the InoInput `type` property.</StepIndicator>

      ```tsx
      const [hidePassword, setVisibility] = React.useState<InputType>('password');

        const togglePasswordVisibility = () => {
          if (hidePassword === 'password') {
          setVisibility('text');
        } else {
          setVisibility('password');
        }
        };
        ```

        ```tsx
        <InoInput
          type={hidePassword}
          label="password"
          outline
          value={password}
          onValueChange={(value) => setPassword(value.detail)}
        >
          <InoIcon
            slot="icon-trailing"
            icon="password"
            clickable
            onClick={() => togglePasswordVisibility()}
          ></InoIcon>
        </InoInput>
        ```

        <StepIndicator step='11'> Add Notifications. Most components have built in styles and themes you can choose
          from. Let's add a different Notification, every time our user signs in.
          If the credentials are too short, we use the property `type="error"` of the `<InoSnackbar>` to indicate a
            failed login attempt. In the case of a succesful login, we can use `type="success"`</StepIndicator>

        ```tsx
        const [showSnackbar, setSnackbar] = React.useState(false);
        const [isValid, setValidity] = React.useState(false);

        const submit = () => {
          setSnackbar(true);
          setValidity(email.length > 0 && password.length >= 8);
          setEmail('');
          setPassword('');

          setTimeout(() => {
          setSnackbar(false);
        }, 5000);
        };

        const ValidateLogin = () => {
          if (isValid) {
          return (
          <InoSnackbar
          action-text="Welcome back!"
          timeout={5000}
        type="success"
        stay-visible-on-hover="false"
      >
        Successfully logged in
      </InoSnackbar>
      );
      } else {
      return (
      <InoSnackbar
        action-text="email and/or password too short"
        timeout={5000}
        type="error"
        stay-visible-on-hover="false"
      >
        Login failed
      </InoSnackbar>
      );
      }
      };
      ```

      ```tsx
      <InoButton
        type="submit"
        onClick={() => {
          submit();
        }}
      >
        Sign In
      </InoButton>;

      {
        showSnackbar ? <ValidateLogin /> : null;
      }
      ```

      Change the types accordingly to your wishes. The timeout property is in _ms_ and indicates when the snackbar
      disappears. The timeout can be disabled by setting it to a negative value.
      More about the properties of the InoSnackbar is documented
      [here](https://elements.inovex.de/version/v7.1.0/?path=/docs/notification-ino-snackbar--playground).
