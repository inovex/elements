# Integration mit Javascript

## Installieren Sie das Paket `@inovex.de/elements`

> Hinweis: Die in diesem Paket enthaltenen Komponenten sind sofort einsatzbereit und werden mit einem Preloader ausgeliefert. Sie müssen die Komponenten nicht manuell importieren, da der Preloader die Komponenten, die Sie verwenden, nachlädt. Wenn Sie also zum Beispiel nur `<ino-button>` verwenden, fordert der Loader nur die Teile des Pakets an, die zum Rendern der Benutzeroberfläche benötigt werden.

Es gibt zwei Wege, die inovex Elements in Ihr Projekt zu integrieren. Entweder können Sie sie mit einem `script` tag von einem zentralen Server einbinden, oder Sie laden sie mit npm herunter und fügen sie in Ihren Build Prozess ein. Es ist Ihre Entscheidung, welche Variante Sie nutzen möchten.

Es gibt einige Faustregeln, wenn Sie nicht wissen, welche Option besser zu Ihrem Projekt passt.

Verwenden Sie das `script` tag, wenn...

- ... Sie die inovex Elements nur ausprobieren wollen.
- ... Sie eine kleine Webseite haben, die keine weiteren Bibliotheken oder Frameworks nutzt.

Verwenden Sie den npm Paket Manager, wenn...

- ... Sie bereits damit begonnen haben, eine Single Page Application zu erstellen.
- ... Sie planen, ihre statische Seite in eine Sigle Page Application zu migrieren.
- ... Sie planen, ein neues Projekt zu starten.

### Installieren mit einem Paket Manager

Fügen Sie das Paket `@inovex.de/elements` mit **npm** or **yarn** zu Ihrem Projekt hinzu:

<CH.Code>
  ```bash npm
  npm install @inovex.de/elements --save
  ```
  ```bash yarn
  yarn add @inovex.de/elements
  ```
</CH.Code>

### Integration mit dem script tag

Um die Komponenten zur Laufzeit zu laden, müssen Sie das folgende Codefragment einfügen.
Stellen Sie sicher, dass das script tag auf jeder Seite gesetzt ist, die der Benutzer auf Ihrer Website besuchen kann.
Für die meisten Websites sollte es ausreichen, den Tag in die Datei `index.html` einzufügen (wählen Sie bei älteren Browsern "Legacy"):
<CH.Code>
  ```html Modern
  <script
    type="module"
    src="https://cdn.jsdelivr.net/npm/@inovex.de/elements@8.0.0/dist/inovex-elements/inovex-elements.esm.js"
  ></script>
  ```
  ```html Legacy
  <script
    nomodule=""
    src="https://cdn.jsdelivr.net/npm/@inovex.de/elements@8.0.0/dist/inovex-elements/inovex-elements.js"
  ></script>
  ```
</CH.Code>

**Abrufen der richtigen Version:** Bitte beachten Sie die Versionsnummer (`8.0.0`) in der obigen Url. Diese Version ist möglicherweise nicht mehr die aktuellste Version. Bitte aktualisieren Sie sie mit der neuesten verfügbaren Versionsnummer auf `elements.inovex.de` während Ihrer ersten Integration.


#### Ich möchte mich nicht auf eine konkrete Version festlegen, können Sie einen Link zur neuesten Version bereitstellen?

Ja, wir bieten einen Link zu unserer neuesten Version an, aber wir empfehlen dringend, diese Links nicht in der Produktion zu verwenden, da die Anwendung jederzeit zusammenbrechen kann. Am besten ist es, eine Versionsnummer zu verwenden, um sicherzustellen, dass die Anwendung nicht unerwartet ausfällt.
Zu Testzwecken oder wenn Sie einfach nur mit den Elementen herumspielen möchten, können Sie die Versionsnummer jedoch jederzeit durch `latest`:

<CH.Code>
```html index.html
<script
  type="module"
  src="https://cdn.jsdelivr.net/npm/@inovex.de/elements@latest/dist/inovex-elements/inovex-elements.esm.js"
></script>
```
</CH.Code>

## Die Komponenten verwenden

Sie können die Komponenten nun auf Ihrer Website genauso verwenden wie jedes andere HTML-Element. Um Ihnen den Einstieg zu erleichtern, haben wir eine [einfache ToDo App](https://stackblitz.com/edit/ino-elements-js-example?embed=1&file=index.js) erstellt, mit der Sie herumspielen können.
